¿Que es esto?

Es demo de un panel de control para utilizar como base para proyectos en angular. Está desarrollado en Angular 9.

A modo de ejemplo tiene un CRUD de tareas y ademas permite la modificación de perfil del usuario loguaedo.

Funcionalidades del sistema

  • Registro de usuarios.
  • Ingreso de usuarios.
  • Recuperar contraseña mediante email.
  • Modificar nombre y contraseña en el perfil de usuario.
  • Agregar tareas.
  • Listar tareas de forma paginada.
  • Buscar tareas por nombre y estado.
  • Modificar el estado de la tarea en el listado.
  • Modificar el estado y el nombre de la tarea en página detalle.
  • Eliminar una tarea.
  • Mostrar hora y fecha de acuerdo a la región configurada en el cliente.

Conocimiento aplicado en el desarrollo de este front-end

  • Comunicación con fuentes de datos externas (API Rest).
  • Implementación de Guards para restringir acceso.
  • Implementación de Lazy Loading.
  • Implementación de Pipes.
  • Utilización de servicios.
  • Utilización de Local Storage para almacenar datos.
  • Agrupación de componentes en módulos.
  • Implementacion de componentes compartidos entre módulos.
  • Implementación de rutas padres e hijas.
  • Utilización programación reactiva mediante rxjs para manejar solicitudes al servidor.
  • Validaciónes nativas de formularios HTML.
  • Utilización de servicios para comunicar componentes.
  • Implementación de Interceptors para manejar la renovación del JWT de forma automatica.
  • Implementación de Interceptors para manejar los errores de las solicitudes al servidor.
  • Utilización de variables de entorno para desarrollo y producción.
  • Segrecación de responsabilidades, componentes se encargan de manejar la vista y los servicios de acceder a los datos.
  • Código escrito de forma limpia y ordenanda, para facilitar la lectura, la mantención y la escalabilidad del proyecto.

¿Como funciona?

Consume un API rest de recursos la cual restringe el acceso mediante un json web token que se obtiene desde un servidor de autenticación.

El json web token tiene un perdiodo de duración, una vez terminado este periodo es necesario renovar el json web token en el servidor de autenticación utilizando un refresh token.

Ambos tokens son proporcionados por el servidor de autenticación cuando un usuario ingresa al sistema mediante el formulario de login.
